# first-lesson-API
Django rest Framework მუშაობს Django-თან ერთად, რომ შევქმნათ ვებ API-ები.
მხოლოდ DRF-ით API-ს შექმნა შეუძლებელია, ყოველთვის საჭიროა Django პროექტი და შემდეგ მასზე დაემატოს DRF.
ამ თავში განიხილება რა მსგავსებები და განსხვავებებია ჩვეულებრივ Django-ს და Django rest Framework-ს შორის. ვიწყებ ჩვეულებრივ ჯანგოს გააქტიურებას...
მივედი უკვე books ზე ანუ გავაკეთე start up books და რადგან ჯანგომ იცნოს დავამატე გადმოწერილ აპებში, ამის შემდეგ გავანახლე ბუქსის მოდელი ასევე შევქმენი ადმინი
გავუშვი პროგრამა და არის ახალი რამ სახელად books დავამატე ეს books ი და დავიწყე view ის შესწორება.
ასევე შევასწორე ორივე urls და დავიწყე template ის გაკეტება.
შევქმენი იგნების ლისტი სადაც მიჩვენებს ყველა ჩემს შექმნილ წიგნს გავუშვი ეს პროგრამა და იმოქმედა შემდეგ გავაკეთე test და ამით დამთავრდა მე-3 თავი.
დავიწყე  djangorestframework ის გადმოწერა, როდესაც გადმოვწერე რატქმაუნდა ჯანგოს გადმოწერილ აპებში დავამატე ასევე დავამატე apis რომელიც ახალი აპია
urls ში დავამატე ახალი apis path ასევე შევქმენი აპის ურლსი და ბოლოს views.py.
ბოლო ნაბიჯი დარჩა და ეს არის serializer-ის შექმნა. Serializer რთულ მონაცემებს, მაგ: queryset-ებს და მოდელების ობიექტებს გარდაქმნის მარტივ ფორმატად, როგორიც არის JSON.
ის ასევე შეძლებს პროცესის შებრუნებასაც — JSON-ის მონაცემების დასერილიალებას, შემოწმებას და Python-ის dictionary-ად გადაქცევას.
Django REST Framework-ის ერთ-ერთი მთავარი უპირატესობა სწორედ serializer-ებია, რომლებიც ამ ყველაფერს ძალიან ამარტივებს ახლა კი დავიწყოთ ამის შექმნა...
დავამატე აპის serializer ი და ჩავწერე კლასები შემდეგ გავუშვი პროგრამა და ყველაფერი წესრიგში იყო დამიწერა ყველაფერი.
